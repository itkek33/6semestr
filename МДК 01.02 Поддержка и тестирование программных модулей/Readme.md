Уровни тестирования:

Компонентное
Интеграционное
Системное
Приёмочное
Тестирование бывает статическим и динамическим

Статическое - без исполнения кода
Динамическое - с исполнением кода
Тесты гугл карты

Функциональные тесты:

Проверка функциональности входа в систему с использованием правильных учетных данных.
Тестирование функции вывода маршрутов.
Проверка правильности отображения данных на странице профиля пользователя.
Тесты производительности:

Измерение времени загрузки веб-страницы при различных уровнях нагрузки.
Оценка времени ответа сервера при обработке большого количества запросов.
Тестирование производительности базы данных при выполнении сложных запросов.
Нагрузочные тесты:

Симуляция одновременного доступа к системе с большим количеством пользователей.
Измерение максимальной нагрузки, которую может выдержать система без снижения производительности.
Тестирование на устойчивость при длительной нагрузке на систему.
Тестирование совместимости:

Проверка работы веб-приложения на различных браузерах (Chrome, Firefox, Safari, Edge).
Тестирование совместимости мобильного приложения с разными версиями операционных систем (iOS, Android).
Проверка работы приложения на разных разрешениях экранов и устройствах.
Позитивные тесты:

Ввод корректных данных в форму регистрации и проверка успешной регистрации пользователя.
Проверка корректной обработки правильного ввода данных в поле поиска.
Проверка правильной работы функциональности добавления нового элемента в список.
Негативные тесты:

Попытка входа в систему с неверным паролем и ожидание сообщения об ошибке.
Ввод некорректного формата электронной почты при регистрации и проверка обработки ошибки.
Попытка удалить маршрут.
Исследовательские тесты:

Изучение внутренней структуры приложения для выявления потенциальных уязвимостей.
Поиск неожиданного поведения приложения при вводе различных комбинаций данных.
Тестирование функциональности, не предусмотренной в документации, для проверки скрытых возможностей или ошибок.
Модульное тестирование:

Тестирование функции вывода маршрута
Тестирование метода вывода карты
Тестирование класса для работы с базой данных пользователей
Интеграционное тестирование:

Тестирование взаимодействия между клиентской и серверной частями приложения
Тестирование интеграции различных модулей системы управления
Тестирование взаимодействия между различными сервисами веб-приложения
Системное тестирование:

Тестирование работы всего приложения
Тестирование производительности системы при большой нагрузке
Тестирование совместимости приложения с различными операционными системами

Отчет о дефекте включает в себя:

Идентификатор
Краткое описание: характер проблемы, границы дифекта, последствия дифекта.
Подробное описание: ожидаемые и фактические результаты.
Влияние - степинь дефект. Критичность и приоритет.
Причины почему дефекты не устраняют

Нехватка времини
Дефект вовсе не дефект
Устранить неисправность слишком рисковано
Это того не стоит
Неэффективный отчет о дефектах

Идентификатор: 1

Краткое описание: Сложение четных чисел происходит неверно

Подробное описание: В ходе сложения двух четных результат больше на 1, чем должно было быть
